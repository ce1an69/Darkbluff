# DarkBluff - 游戏设计文档

DarkBluff 是一款 CLI/TUI 文字推理游戏。玩家扮演一只拥有异色瞳的猫——左眼看到影子世界，右眼看到表面世界——通过对比两个世界的信息（表面恒真、影子恒假）进行逻辑推理，侦破一个个案件。

- **类型**: CLI/TUI 文字推理游戏
- **美学风格**: 黑暗、悬疑、混沌- **实现**: Rust + ratatui，单二进制分发，剧情数据与代码完全分离

> 本文档按主题拆分。本页是**概览与导航枢纽**；各主题的完整规格在下方链接的子文档中，按需阅读（渐进式披露）。

## 1. 项目概览

- **名称**: DarkBluff
- **类型**: CLI/TUI 文字推理游戏
- **核心概念**: 通过对比表面世界（真话）与影子世界（谎言）的信息进行逻辑推理
- **美学风格**: 黑暗、悬疑、混沌- **主角**: 一只拥有异色瞳的猫——左眼可以看到影子世界

### 1.1 语言策略

- 首发版本只支持中文，不设计多语言/i18n 数据结构
- 剧情文本、UI 展示文本、话题标签均使用中文维护
- 代码标识符、文件名、指令名、数据 ID 保持英文 snake_case，便于实现和长期维护

---

## 2. 世界观设定

### 2.1 表面世界

- 拟人化动物社会的"正常面貌"
- 所有角色说的都是真话
- 表面平和，实则暗流涌动

### 2.2 影子世界

- 同一世界的黑暗/扭曲面
- 所有角色无意识地说谎（他们自己不知道在说谎）
- 氛围更加混乱与黑暗
- 灵感来自《大异兽》的洞穴世界：混沌、暴力常态化、阶级压迫

### 2.3 主角

- 一只拥有「双瞳」的猫
- 左眼看到影子世界，右眼看到表面世界
- 可以自由切换视角，从两个世界收集信息
- 利用表面/影子之间的信息差推理出真相

---

## 3. 核心玩法速览

游戏在一个章节（案件）内的循环：**观察场景 → `gaze` 切换影子视角获取对比信息 → `ask` 收集线索 → 分析表面/影子矛盾 → `judge` 审判 → 完成必要审判后自动推进剧情**。

核心推理机制是**二元真值**：表面世界信息恒为真命题，影子世界信息恒为假命题（取反即得真相），玩家结合两侧约束缩小可能性。难度通过约束数量与推理链深度提升，而非破坏二元真值。

> 完整机制、作者守则、难度轴详见 [gameplay.md](gameplay.md)。

## 4. 指令速查

| 指令    | 语法                 | 用途                                          |
| ------- | -------------------- | --------------------------------------------- |
| `ask`   | `ask [目标] [话题?]` | 从角色收集信息；不指定话题时弹出话题菜单 |
| `judge` | `judge [目标]`       | 审判某角色（章级操作，不要求在场），触发审判剧情；无参数则列出本章未审判的角色 |
| `move`  | `move [目的地]`      | 在场景之间移动                                |
| `gaze`  | `gaze`               | 切换左/右眼（影子/表面视角）                  |
| `map`   | `map`                | 打开章节树/检查点地图，可选择已经历过的 checkpoint 回滚 |
| `note`  | `note`               | 查看玩家已见过的文本记录（对话/叙事/审判剧情）  |
| `help`  | `help [指令?]`       | 列出指令或查看某指令用法                      |
| `quit`  | `quit`               | 自动保存后退出                                |

> 交互流程示例与运行时错误处理详见 [commands.md](commands.md)。

## 5. 叙事速览

每个章节是一个 15–30 分钟的独立案件，多章节构成连续主线，最终揭示表面/影子世界的本质。`judge` 审判没有选项、没有对错——审判了哪些角色累积影响章节跳转、导向多个不同的结局分支；其中由特定审判序列到达的真相结局揭示世界本质。

游戏采用**单存档自动保存**，通过 `map` 打开的章节树/检查点地图支持分支探索：玩家可选择已经历过的 `chapter_start` 或 `before_judgment` checkpoint 回滚。所有探索记忆记录在 append-only 的 `discovered` 中，回滚不丢失。

> 章节/结局结构、重玩与检查点、启动与标题界面详见 [narrative.md](narrative.md)。

## 6. 数据架构速览

叙事内容与代码完全分离：结构化数据用 YAML，大段叙事文本用 Markdown，YAML 用相对路径引用 Markdown，引擎运行时加载。场景/角色为全局资源，章节通过 ID 引用并可选覆盖。所有内容实体有唯一的 `id`（同时作展示引用、存档外键与条件标记，发布后冻结）。

## 7. 技术速览

Rust 实现，核心库、CLI 与 TUI 已拆为 workspace crate；TUI 使用 ratatui + crossterm 渲染。serde 读写数据/存档，`include_dir` 发布内嵌数据为后续 TODO。当前 TUI 为同步事件循环，支持短过渡动画 tick。

> 技术栈、项目结构、架构分层、技术约束详见 [architecture.md](architecture.md)。

## 8. 设计原则

| 原则                  | 说明                                                                             |
| --------------------- | -------------------------------------------------------------------------------- |
| 数据驱动              | 叙事/角色/对话与代码完全分离，新增章节无需修改代码                               |
| 结构与内容分离        | YAML 管理结构和逻辑（含可见性/解锁条件），Markdown 存储叙事文本                  |
| 全局资源 + 章节可覆盖 | 场景、角色为全局资源，章节通过 ID 引用并可选覆盖描述                            |
| 按角色拆对话          | 每章每角色一个 .md 文件，避免单文件过大                                          |
| 路径字符串引用        | 所有文本引用为相对路径，引擎运行时加载                                           |
| 内容引擎无状态        | 只做加载和查询，游戏状态由引擎层管理                                             |
| 单一 id               | 每个实体只有一个语义化 id，同时作引用、存档外键与条件标记，发布后冻结不得改名    |
| 条件表达统一          | 话题解锁和章节跳转使用同一套扁平条件表达式（id）                                 |
| 存储事实为权威        | clues/views/judgments 直接存储；检查点记录数组长度，回滚按长度截断，不做事件回放 |
| 探索记忆 append-only  | `discovered` 记录所有到过的章节/结局/审判，任何回滚都不截断                      |
| 笔记快照优先          | note 展示玩家实际见过的文本，不随剧情更新漂移                                    |
| 存档兼容与健壮性      | 未知字段忽略、缺失字段默认值、版本迁移、原子写入 + `.bak` 备份                   |
| 事件驱动渲染          | 无动画时只响应输入；动画期间才启用 tick 重绘                                     |
| 单二进制分发          | 编译后为单个可执行文件，游戏数据可内嵌或外置                                     |
| 内容可校验            | `darkbluff check` 可在不启动 TUI 的情况下校验数据文件                            |
| 动画可关闭            | `--no-motion` 或配置项关闭/缩短过渡动画                                          |
| 中文首发              | 首发不做 i18n，所有玩家可见内容默认中文                                          |
