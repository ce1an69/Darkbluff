# 指令系统

> 属于 DarkBluff 设计文档。主索引见 [design.md](design.md)。

## 指令一览

| 指令    | 语法                 | 用途                                         |
| ------- | -------------------- | -------------------------------------------- |
| `ask`   | `ask [目标] [话题?]` | 从角色收集信息；不指定话题时弹出话题菜单     |
| `judge` | `judge [目标]`       | 审判某角色（触发审判剧情）；无参数则列出本章未审判角色 |
| `move`  | `move [目的地]`      | 在场景之间移动                               |
| `gaze`  | `gaze`               | 切换左/右眼（影子/表面视角）                 |
| `map`   | `map`                | 打开章节树/检查点地图，可选择已经历过的 checkpoint 回滚 |
| `note`  | `note`               | 查看玩家已见过的文本记录（对话/叙事/审判剧情） |
| `help`  | `help [指令?]`       | 列出指令或查看某指令用法                     |
| `quit`  | `quit`               | 自动保存后退出                               |

**指令分类**：

- 核心交互：`ask` / `judge` / `move` / `gaze`
- 信息查看：`map` / `note` / `help`
- 流程控制：`quit`

## 关键语义

- **`ask` 目标**: 当前场景中在场的角色 ID（角色在场判定见 [data-formats.md](data-formats.md)「字段说明 → characters.appears_in」）。
- **重复询问同一话题**: 再次 `ask` 已问过的话题仍正常展示对话（允许随时回顾），但线索不重复收集（`collected_clues` 为 set）、快照不重复写入（已存在则跳过）。`note` 始终基于首次查看时的快照。重复询问时不提供"已问过"标记——回顾信息是推理游戏的核心操作，不应被暗示为"冗余行为"。
- **`move` 与视角**: `move` 切换场景时**保留当前视角**（`current_world` 不变），玩家无需重新 `gaze`。启动校验保证每个被章节引用的场景同时有 surface 和 shadow 描述（见 [content-engine.md](content-engine.md)），因此运行时不存在视角缺失的情况。
- **场景描述面板**: 场景描述面板始终可见；`move` 到达新场景后、`gaze` 切换视角后，自动刷新当前场景当前视角的描述文本。
- **`move` 无参数**: `move` 不带目的地时，列出当前场景可达连接中的可去场景供选择（含引擎自动补全的反向连接，与 `ask` 无话题弹菜单对称）；带参数时校验目的地在可达连接中，否则提示"你现在无法前往那里。"。
- **视角指示器**: UI 顶部常驻显示当前章节标题、当前场景名、当前视角（`👁 右眼·表面` / `👁 左眼·影子`），保证推理时玩家始终知道自己在哪只眼。
- **初始视角**: 新游戏首章与每次进入新章节时，视角默认为 `surface`（右眼·表面）；玩家随后用 `gaze` 主动切换（见 [data-formats.md](data-formats.md)「自动推进章节」）。
- **自动推进章节**: `judge` 后若本章 `required_judgments` 已全部完成，系统立即评估章节跳转并进入下一章；若当前章为终章，则记录结局、展示 `outro`（若有）并进入结局界面。详见 [data-formats.md](data-formats.md)「章节元数据 → 自动推进章节」。
- **破坏性操作的确认**: `map` 选择 checkpoint 回滚会丢弃回滚点之后的当前流程进度，执行前需二次确认；Esc 可取消（见 [save-system.md](save-system.md)）。
- **`ask` 无参数**: `ask` 不带目标时，先列出**当前场景在场角色**（由 `characters[].appears_in` 含当前场景推导），选择角色后再列出该角色可问的话题菜单（**已解锁话题，并按当前视角过滤**——只列出在当前 `current_world` 下有对话版本的话题；只有另一侧版本的 single-world topic 不进菜单，避免选中后撞「这一侧无从问起」）。即两步菜单：角色 → 话题。
- **`judge` 无参数**: `judge` 不带目标时，列出**本章尚未审判的角色**（即 `judgments.yaml` 中定义了审判点、且该角色尚未被审判的），选择后触发该角色的审判剧情。`judge` 是章级操作，不要求目标角色在当前场景在场（与 `ask` 的场景级在场检查不同）。
- **`map` 的 checkpoint 范围**: `map` 展示玩家已经历过的 `chapter_start` 与 `before_judgment` checkpoint。选择 `chapter_start` 等同于回到该章开头；选择 `before_judgment` 等同于回到某次审判前。跨章选择会销毁回滚点之后的当前流程进度，但 `discovered` 保留。

## 交互流程

**ask 流程**（菜单 / 直接两种入口）：

```
> ask wolf
[话题列表]
1. 昨晚的行踪
2. 与受害者的关系
3. 那把刀
> 1
灰狼："我整晚都在酒馆……"
[对话已记录到 note，相关线索已收集]

> ask wolf whereabouts        # 直接指定话题
灰狼："我整晚都在酒馆……"
```

直接输入话题时使用话题 `id`（英文 snake_case，如 `whereabouts`）；菜单模式下按数字编号选择，展示中文 `label`。不支持输入中文 label 匹配话题。

直接输入未解锁的话题时（`ask wolf secret`），提示「你还没有足够线索。」（见运行时错误表）。

**gaze 流程**：

```
> gaze
[已切换到左眼 - 影子世界]
场景描述发生变化…同一场景的扭曲版本
```

**judge 流程**（审判前自动建立检查点）：

```
> judge wolf
[已自动建立检查点 ckpt_before_judge_wolf]
[触发灰狼的审判剧情]
灰狼被带上审判台……（result 文本）
[已记录：审判过灰狼]
[本章必要审判尚未全部完成，继续探索]
```

若本章必要审判已全部完成：

```
> judge crow
[已自动建立检查点 ckpt_before_judge_crow]
[触发乌鸦的审判剧情]
[已记录：审判过乌鸦]
[必要审判已完成，自动推进章节]
[评估章节跳转条件 → 进入下一章]
```

若为终章且含 `outro`：

```
> judge wolf
[已记录：审判过灰狼]
[必要审判已完成，达成结局]
（展示 outro 结局收尾文本……）
[按任意键继续]
→ 进入结局界面（结局名 + 已发现结局 X/Y + 返回标题）
```

若 wolf 已被审判过：

```
> judge wolf
[你已经审判过他了。（可用 map 回到审判前 checkpoint）]
```

**map 流程**（章节树 / checkpoint 回滚）：

```
> map
[章节树 / checkpoint 地图]
> 选择 ckpt_before_judge_wolf
[回滚会丢弃该 checkpoint 之后的当前流程进度，discovered 保留。确认？]
> y
[已回到审判前]
```

## 运行时错误处理

内容缺失或玩家输入无效时，必须以可读提示返回 UI，不允许 panic。

| 场景 | 处理方式 |
|------|----------|
| `ask` 的目标不在当前场景 | 提示"这里没有这个角色。" |
| `ask` 通过菜单访问 | 未解锁话题不展示在菜单中 |
| `ask [目标] [话题]` 直接输入未解锁话题 | 提示"你还没有足够线索。" |
| `ask` 话题在当前世界无文本（单世界话题） | 提示"这个话题在这一侧无从问起。" |
| `move` 的目标不在当前场景可达连接中 | 提示"你现在无法前往那里。" |
| `move` 无参数且当前场景无可达连接 | 提示"没有可以前往的地方。" |
| `judge` 的目标角色不在本章 | 提示"现在还无法审判他。" |
| `judge` 的目标角色在本章但无审判点 | 提示"现在还无法审判他。" |
| `judge` 的目标角色已被审判过 | 提示"你已经审判过他了。（可用 map 回到审判前 checkpoint）" |
| `judge` 后必要审判未完成 | 回到 `Exploring`，不推进章节 |
| `map` 没有可选 checkpoint | 提示"还没有可以回到的节点。" |
| `map` 选择的 checkpoint 已失效 | 提示"这个节点已经无法回到。"并刷新地图 |
| `note` 快照文件缺失 | 显示"该记录快照缺失"，不回退到最新剧情文本 |
| 存档引用的内容已删除 | 跳过该条记录，并在加载报告中记录 warning |

**输入解析层（语法错误）**: 在内容/状态层错误之外，指令解析层的异常也需友好处理，不得 panic：

| 场景 | 处理方式 |
|------|----------|
| 未知指令（拼写错误等） | 提示"未知指令：{输入}。输入 help 查看可用指令。" |
| `help [未知指令名]` | 提示"未知指令：{输入}。输入 help 查看可用指令。" |
| 空输入 | 忽略，不报错，重新等待输入 |
| 指令多余参数 | 提示"该指令不需要参数。"或忽略多余参数 |
| 大小写 | 指令名与目标/话题 ID 不区分大小写（规范化为小写后匹配） |
| 多余空白 | 容忍参数间多空格，按空白分词 |

**输入辅助**: 输入框支持 Tab 补全指令名/目标/话题 ID，↑/↓ 调出历史指令。补全候选范围与指令上下文一致：

| 补全上下文 | 候选范围 |
|------------|----------|
| 指令名 | 全部 8 个指令 |
| `ask` 目标 | 当前场景在场角色 |
| `ask [目标]` 话题 | 该角色当前已解锁、且在当前视角下有版本的话题 id |
| `judge` 目标 | 本章尚未审判的角色 |
| `move` 目的地 | 当前场景可达连接（含引擎补全的反向连接） |
| `help` 指令 | 全部 8 个指令 |

**多候选处理**: 当 Tab 候选有多个匹配时，补全到最长公共前缀并展示候选列表；再次 Tab 循环选择候选项。仅有唯一匹配时直接补全。

## 菜单取消

所有选择菜单（ask 角色/话题选择、judge 角色选择、move 目的地选择、map checkpoint 选择）均支持 **Esc 取消**，回到 `Exploring` 状态，不产生任何副作用：

- `ask` 的角色或话题选择菜单中取消 → 回到 `Exploring`，不收集线索、不写快照
- `judge` 的角色选择菜单中取消 → 回到 `Exploring`，不创建检查点（检查点在执行审判时才创建）
- `move` 的目的地选择菜单中取消 → 回到 `Exploring`，不移动
- `map` 的 checkpoint 选择菜单中取消 → 回到 `Exploring`，不回滚
- 破坏性操作的确认对话框（map checkpoint 回滚 / 覆盖存档）中取消 → 回到之前状态

启动校验负责发现内容数据本身的问题（见 [content-engine.md](content-engine.md)）；运行时错误处理负责保护玩家存档和异常输入。

## 菜单状态下的输入隔离

进入选择菜单状态（角色选择 / 话题选择 / 目的地选择）后，仅接受：

- 数字编号选择菜单项
- Esc 取消回到 `Exploring`
- ↑/↓ 导航候选项

其他输入静默忽略，不处理不报错。玩家须先 Esc 退出菜单再输入新指令。
